Spec 018 — Block Indicator Facility · UX Mockups

Illustrative mockups showing the new Facilities and Channels surfaces in the Railroad tab plus the supporting pickers. These are static design references; final styling will use the existing Bowties component set.

Tab labels match the current app (Config · Bowties · Railroad). The Railroad tab has no sidebar in this slice — facility cards already serve as their own detail view, the Channels section sits below on the same scrolling page, and adding sidebar nav doesn't earn its keep at this scale. Traffic Monitor is a separate window and not a tab.

7. "Add channel" sub-picker — Direct Lamp Control row

Lists unclaimed, constraint-compatible Direct Lamp Control rows (FR-018, FR-030). Rows where Lamp Selection = "Used by Mast" are hidden, not greyed out. On confirm, Bowties atomically creates the channel, binds it to the row, and binds it to the slot.

9. Edge states — no compatible channels / no hardware target

Slot operations are disabled with hover tooltips that say what's missing in role terms. Both "Select" and "Add" can be unavailable at the same time; the slot still accepts no-op state (Acceptance Scenario 2, Edge Cases). Richer slot-hint UX (recommending which specific hardware to add) is a vision-level concern, not a spec 018 deliverable.

10. "Unsaved Changes" dialog — mixed node + facility edits

Confirm shown when the user tries to close, disconnect, or exit with edits pending across both node-level surfaces (CDI config, bowtie metadata, connector selections, offline drafts) and the new facility/channel surfaces from spec 018. The breakdown is one bullet per non-empty bucket — quiet when nothing changed, dense only when something did. Destructive action is Fluent danger red; the orange ⚠ glyph labels the dialog as a warning, the button color names the action. Esc, overlay click, and the bare × all map to Cancel.
