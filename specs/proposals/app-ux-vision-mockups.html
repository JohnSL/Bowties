1. App Shell — Workspace Toggle

The top-level navigation replaces today's Config/Bowties segmented control with Wiring/Railroad workspaces.

  • Click the Wiring/Railroad toggle to switch workspace views.
  • Click "Board Settings" in the sidebar to see board-global (non-channel) configuration.
  • Click "Raw CDI" in the sidebar to expand the full CDI segment tree (escape hatch — same as today's Config experience).
  • The Daughter Board combo box at the top of the connector view sets the board type (Tier 1). Changing it removes existing channels.
  • Pins 5–8 show "Assign type →" — clicking these leads to the channel type picker in Mockup 2 (Tier 2, per-pin).
  • Click "▾ details" on a channel row to expand managed/unmanaged settings inline.

2. Wiring — Channel Type Application (Per-Pin)

For boards where each pin can become a different channel type (Tier 2).

  • Entry point: click "Assign type →" on an unconfigured pin in Mockup 1 (pins 5–8).
  • Card-based channel type picker shows compatible types with pin count and which pins will be claimed.
  • After selection: constraint confirmation shows which managed fields were set.
  • Wiring settings (unmanaged) remain freely adjustable below.
  • "Create Channel" commits the channel; it immediately appears in both workspaces.

3. Railroad — Facility Comprehension View

The primary debugging surface — the facility bowtie visualized.

  • Inputs (left): channels feeding in (occupancy, position feedback).
  • Logic (center): behavior decisions, where they execute, current evaluation state.
  • Outputs (right): channels the facility drives (signal aspects, turnout commands).
  • Each channel shows live state when connected.
  • Click any element to jump to its backing hardware in the Wiring workspace.

4. Template Apply — Side Panel Workflow

Template apply opens as a side panel alongside the existing layout view.

  • Requirements are color-coded: green (matched), red (no match yet).
  • Unmapped requirements can be left pending — partial application creates an incomplete facility.
  • "Create new channel inline" lets the user create hardware channels without leaving the template flow.
  • The "Next" button advances even with pending slots.

6. Template Library

Browsable by behavior category.

  • Category filter tabs (Signaling, Detection, Turnout Control, Panel, Lighting).
  • Each card shows: name, one-sentence purpose, required channel types, and logic capacity.
  • Selecting a template starts the side-panel apply workflow (Mockup 4).

Behavior Templates

ABS 3-Aspect Signal Block

Signaling

Standard ABS block signaling with approach-lit 3-aspect logic. Handles occupied, approach, and clear indications.

Needs: 2× occupancy, 1× signal, logic capacity

Passing Siding with ABS

Signaling

Complete passing siding with detection, two turnouts, and 3-aspect signaling at both ends. Includes approach lighting.

Needs: 3× occupancy, 2× turnout, 2× signal, logic capacity

CTC-Controlled Turnout

Panel / CTC

Dispatcher-controlled turnout with indication, local override via pushbutton, and interlocking with adjacent blocks.

Needs: 1× turnout, 1× button, 2× occupancy, logic capacity

Approach Lighting

Lighting

Automatically illuminates a signal head only when a train is in the approach block. Reduces power consumption and adds realism.

Needs: 1× occupancy, 1× signal, logic capacity

  • Board cards show model, vendor, estimated cost, and connector assignments.

  • “Modify Answers” returns to the wizard to change inputs.
